What zub does

The product is intentionally narrow: install a binary from the registry, make sure the required Zig version exists, build it, and drop the executable where your shell already expects user binaries.

01

Read the registry

zub downloads the package list from the public registry and resolves the package entry by name.

02

Prepare the toolchain

It ensures zvm is available, detects the package Zig version, and installs that version when needed.

03

Build and install

The package is built from source and the resulting binary is copied into $HOME/.local/bin.

Install zub

The intended bootstrap flow is a single shell command. The installer provisions zvm, installs a bootstrap Zig version, builds zub, and installs it locally.

Bootstrap command

curl -fsSL https://yafb.net/zub.zig/install.sh | bash

Default install targets

zub binary $HOME/.local/bin/zub
source cache $HOME/.cache/zub/src
zvm home $HOME/.zvm

Use zub

Once installed, package installation is intentionally small. Ask for a binary by name and let zub do the registry lookup, toolchain selection, build, and final install.

Install a package

zub install pbm

Operational notes

The current implementation keeps the contract simple and explicit. It focuses on reproducible source builds of Zig binaries with version-aware toolchain provisioning.

How is the repository resolved?

The current package URL contract assumes registry entries use the pattern /packages/<owner>/<repo>/, which maps directly to a GitHub repository.

What if the package does not declare a Zig version?

If minimum_zig_version is not present in build.zig.zon, zub falls back to master.

Where does the binary go?

The installed executable is copied into $HOME/.local/bin. Make sure that directory is present in your shell PATH.
